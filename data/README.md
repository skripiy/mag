# Дані системи

Домен демонстрації — гаряча лінія **гуманітарної організації** (підтримка ВПО та
постраждалих від бойових дій). Відповідно до архітектури розділяються два типи
даних (підрозділ 2.7).

## `kb/` — база знань (RAG-корпус)

Довідкові матеріали організації (FAQ/інструкції/регламенти). **PII не містить** →
можна тримати реальний контент. Індексується у pgvector:

```bash
python -m app.indexing data/kb
```

Документи:
- `groshova_dopomoga.md` — грошова (багатоцільова) допомога
- `reestratsiya_vpo.md` — реєстрація ВПО, довідка, поновлення виплат
- `humanitarni_nabory.md` — продуктові/гігієнічні набори, видача, доставка
- `prytulok_zhytlo.md` — тимчасовий прихисток, місця проживання, оренда
- `psyhosocialna_pidtrymka.md` — психологічна підтримка
- `pravova_dopomoga.md` — правова допомога, відновлення документів, компенсації

## `requests/hotline_requests.jsonl` — тестовий потік звернень

Реалістичні формулювання звернень для функціонального тестування, оцінки якості
RAG (context precision/recall) та демонстрації анонімізації (Розділ 4).

> **PII у датасеті — синтетичні** (вигадані імена, номери телефонів, РНОКПП/ІПН,
> адреси коректного формату). Реальні персональні дані не використовуються.

Поля кожного запису:

| Поле | Опис |
|---|---|
| `id` | ідентифікатор звернення |
| `text` | текст звернення (як від користувача) |
| `category` | цільова тема (або `ambiguous` / `out_of_scope`) |
| `expected_doc` | очікуваний релевантний документ бази знань (або `null`) |
| `has_pii` | чи містить синтетичну PII (для перевірки анонімізації) |
| `note` | примітка (тип PII, багатотемність тощо) |

Датасет навмисно містить: чіткі однотемні запити, багатотемні звернення, розмиті
(`ambiguous`) та поза межами компетенції (`out_of_scope`) — останні перевіряють
шлях «недостатньо інформації в базі знань».
